# Product Modules

WechatAgent 的模块划分按运营业务域，而不是按 MCP 工具命名。MCP 是能力层，产品模块是业务层。

## 一级模块

```text
工作台
用户运营
微信群运营
朋友圈运营
内容资产
系统策略
AI Command Center
任务与日志
账号与系统
```

## 工作台

目标：判断系统是否正常运行，并提供关键模块入口。

包含：

- 微信账号在线状态
- 今日消息数
- Agent 回复数
- 已纳管对象数量
- 待执行任务
- 失败任务
- 最近运营事件

不包含：

- 大量配置项
- 完整列表管理
- 长表格
- 多步骤操作

## 用户运营

目标：对单个微信好友做长期、个性化、自动化运营。

当前已实现：

- 搜索导入好友
- `normal / managed` 状态
- 自然语言运营备注
- 用户运营域配置：长期目标、方法论、工作流、工具边界、自动化策略、复盘规则和运行参数
- AI 生成画像
- 账号级运营方法配置
- AI 生成运营方法
- 自由标签、阶段、意向、承诺和跟进策略
- 运营大脑长期记忆
- 可配置用户运营状态机
- NextBestAction 决策层
- 统一发送网关，私聊回复和 follow-up 都强审查
- 运行参数硬约束：频控、每日触达、Review 阈值、跟进上限和任务过期
- 用户运营 Soul Prompt 和 Prompt Template 独立编辑
- 独立评审评分约束
- 会话记录
- managed 好友自动回复
- 跟进任务

后续补充：

- 更细的产品知识检索和排序
- 更完整的用户反应情绪分类
- Playbook 效果统计和自动优化建议

第一优先级仍然是用户运营，因为它最容易验证 Agent 是否真正创造价值。

## 微信群运营

目标：对微信群进行分析、线索发现和可控运营。

第一阶段建议只做：

- 群列表
- 群基础画像
- 群消息监听
- 群话题总结
- 重点消息识别
- 潜在线索发现
- 回复草稿建议

默认不做：

- 自动群内发言
- 自动邀请/移除成员
- 自动修改群公告

群运营比私聊风险更高。第一版群 Agent 应以“分析和建议”为主，后续再按白名单群、关键词、@ 我等条件开放自动发言。

## 朋友圈运营

目标：管理内容计划、草稿、素材、定时发布和发布结果。

包含：

- 朋友圈内容计划
- AI 生成草稿
- 图片/视频/链接素材选择
- 定时发布队列
- 发布历史
- 评论/点赞跟进

默认策略：

- 先生成草稿
- 默认需要人工确认或明确自动规则
- 不允许 Agent 随机发布没有来源的内容

## 运营知识库

目标：为用户运营 Agent 提供可路由、可审计、可渐进加载的运营知识，而不是把全部资料直接塞进上下文。

当前生产级形态包含：

- 文本/Markdown 导入预览
- AI 自动生成文档目录、知识包、知识切片和证据块
- 文档目录 `operation_knowledge_documents`
- 主题知识包 `operation_knowledge_items`
- 运行时按需打开的知识切片 `operation_knowledge_chunks`
- Agent 内部工具：`knowledge.list_catalog`、`knowledge.search`、`knowledge.open_slice`、`knowledge.open_evidence`
- 命中测试展示工具轨迹、选中切片、缺失知识和覆盖度
- Agent 使用日志记录 routeResult、toolTrace、回复和 Review 结果

默认策略：

- 用户运营优先接入。
- Mongo 结构化目录 + LLM 自主工具规划，不依赖向量库。
- 不使用固定分类枚举；知识类型、业务上下文和适用场景由 AI 生成，人可以在 UI 中自然语言修改。
- 没有知识依据时，Agent 只能保守回应，不得回答产品承诺、案例、报价、效果保证。

## 内容资产

目标：让 Agent 有稳定知识来源和表达边界，避免每次临时生成导致风格漂移。

包含：

- 产品资料
- 服务介绍
- 报价说明
- 结构化产品知识
- 安全可说事实
- 禁止承诺
- FAQ
- 案例库
- 朋友圈素材
- 图片/视频素材
- 话术库
- 禁用表达
- 品牌语气

内容资产是用户运营、群运营和朋友圈运营共用的知识层。

## 系统策略

目标：集中管理跨模块的后台总控 Agent、方法论生成 Agent 和全局 Prompt Pack。

包含：

- 全局 Agent 规则
- 后台管理 Agent Soul Prompt
- 方法论生成 Prompt
- Prompt Stack v2
- Prompt 版本
- Management Agent 权限

不包含：

- 用户运营的具体目标、工作流、方法论和用户 Prompt
- 群运营的具体目标、工作流、方法论和群 Prompt
- 朋友圈运营的具体目标、工作流、方法论和朋友圈 Prompt

这些内容应在对应业务频道内维护，避免把完全不同的运营逻辑混在一个“Agent 策略”页里。

Prompt Stack v2 应作为独立子频道展示：

- Soul Prompt：稳定人格
- System Contract：输出格式、事实边界和安全约束
- Policy：自动化边界、风险分级、频控和禁忌
- Task Template：画像、回复、方法生成等任务模板
- Review：独立评审和拦截规则
- Methodology Generator：运营方法论生成/优化模板

示例默认策略：

```text
用户运营：managed 好友允许自动回复
微信群运营：默认只生成分析和草稿
朋友圈运营：默认生成草稿，发布需规则或确认
```

## AI Command Center

目标：提供 AI 时代的总后台操作入口，让内部操作员通过自然语言管理整个项目。

包含：

- Management Agent 对话
- 系统执行计划
- 工具调用状态
- 待确认高风险动作
- 自然语言查询和配置
- 跳转到具体好友、群、朋友圈计划、任务和日志

典型指令：

```text
把 xx 加入运营列表
发送 xxx 给好友 xxx
新建一个群拉 xxx 进群
查看今天失败的任务
帮我生成本周朋友圈运营计划
```

AI Command Center 不替代业务模块。它负责跨模块调度，具体对象管理仍落在用户运营、微信群运营、朋友圈运营、内容资产和 Agent 策略中。

## 任务与日志

目标：统一管理系统执行过程，保证长期运行可追踪、可复盘。

包含：

- Agent 决策日志
- MCP 调用日志
- 消息发送日志
- 任务执行日志
- 失败重试
- 定时任务
- 异常告警

各业务模块可以展示局部日志，但总账在任务与日志模块。

## 账号与系统

目标：管理微信账号、MCP、模型和系统配置。

包含：

- 微信账号列表
- 登录状态
- MCP Key 配置
- Webhook 配置
- 模型配置
- MongoDB 状态
- 操作员账号
- 权限配置
