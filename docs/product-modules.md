# Product Modules

WechatAgent 的模块划分按运营业务域，而不是按 MCP 工具命名。MCP 是能力层，产品模块是业务层。

## 一级模块

```text
工作台
用户运营
微信群运营
朋友圈运营
内容资产
Agent 策略
任务与日志
账号与系统
```

## 工作台

目标：判断系统是否正常运行，并提供关键模块入口。

包含：

- 微信账号在线状态
- 今日消息数
- Agent 回复数
- 已纳管对象数量
- 待执行任务
- 失败任务
- 最近运营事件

不包含：

- 大量配置项
- 完整列表管理
- 长表格
- 多步骤操作

## 用户运营

目标：对单个微信好友做长期、个性化、自动化运营。

当前已实现：

- 搜索导入好友
- `normal / managed` 状态
- 自然语言运营备注
- AI 生成画像
- 会话记录
- managed 好友自动回复
- 跟进任务

后续补充：

- 用户标签
- 用户分层：新客、老客、高意向、沉默、成交
- 跟进节奏
- 最近承诺记录
- 禁止重复触达规则
- 客户阶段变化日志

第一优先级仍然是用户运营，因为它最容易验证 Agent 是否真正创造价值。

## 微信群运营

目标：对微信群进行分析、线索发现和可控运营。

第一阶段建议只做：

- 群列表
- 群基础画像
- 群消息监听
- 群话题总结
- 重点消息识别
- 潜在线索发现
- 回复草稿建议

默认不做：

- 自动群内发言
- 自动邀请/移除成员
- 自动修改群公告

群运营比私聊风险更高。第一版群 Agent 应以“分析和建议”为主，后续再按白名单群、关键词、@ 我等条件开放自动发言。

## 朋友圈运营

目标：管理内容计划、草稿、素材、定时发布和发布结果。

包含：

- 朋友圈内容计划
- AI 生成草稿
- 图片/视频/链接素材选择
- 定时发布队列
- 发布历史
- 评论/点赞跟进

默认策略：

- 先生成草稿
- 默认需要人工确认或明确自动规则
- 不允许 Agent 随机发布没有来源的内容

## 内容资产

目标：让 Agent 有稳定知识来源和表达边界，避免每次临时生成导致风格漂移。

包含：

- 产品资料
- 服务介绍
- 报价说明
- FAQ
- 案例库
- 朋友圈素材
- 图片/视频素材
- 话术库
- 禁用表达
- 品牌语气

内容资产是用户运营、群运营和朋友圈运营共用的知识层。

## Agent 策略

目标：集中管理自动化边界和行为规则。

包含：

- 全局 Agent 规则
- 用户运营策略
- 群运营策略
- 朋友圈策略
- 回复风格
- 触发条件
- 频控规则
- 禁忌规则
- 自动化边界

示例默认策略：

```text
用户运营：managed 好友允许自动回复
微信群运营：默认只生成分析和草稿
朋友圈运营：默认生成草稿，发布需规则或确认
```

## 任务与日志

目标：统一管理系统执行过程，保证长期运行可追踪、可复盘。

包含：

- Agent 决策日志
- MCP 调用日志
- 消息发送日志
- 任务执行日志
- 失败重试
- 定时任务
- 异常告警

各业务模块可以展示局部日志，但总账在任务与日志模块。

## 账号与系统

目标：管理微信账号、MCP、模型和系统配置。

包含：

- 微信账号列表
- 登录状态
- MCP Key 配置
- Webhook 配置
- 模型配置
- MongoDB 状态
- 操作员账号
- 权限配置
