# Development Roadmap

## Phase 1: 用户运营闭环

目标：让私聊好友运营稳定、可控、可追踪。

已完成：

- Rust 后端
- React 后台
- MongoDB 数据层
- MCP 调用
- DeepSeek 模型调用
- 好友导入
- `normal / managed`
- 运营备注
- 画像生成
- 账号级运营方法
- AI 生成运营方法
- 自由标签、阶段、意向、承诺和跟进策略
- 运营大脑长期记忆
- 产品知识安全边界
- 独立 Review Agent 评分约束
- 自动回复
- 跟进任务
- 日志记录

下一步建议：

- 分页和搜索增强
- webhook 载荷样例和测试工具
- 客户阶段变化日志
- 跟进频率和重复触达规则
- Review Agent 测试集和运营质量基准

## Phase 2: 内容资产 + Agent 策略

目标：让 Agent 的内容、人格和行为边界可配置。

要做：

- 内容资产库
- FAQ/产品资料/案例库
- 品牌语气规则
- Agent Soul Prompt
- Prompt 版本管理
- 运营方法 Playbook 版本管理
- 运营大脑方法论配置
- 产品知识结构化管理
- 决策复盘评分
- 禁用表达
- Agent 策略配置
- 频控规则
- 自动化边界配置

交付标准：

- Agent 回复能引用内容资产。
- Agent 有稳定 Soul，不因单次调用发生风格漂移。
- 运营人员能配置 Agent 行为风格。
- 自动化边界不再硬编码在 prompt 里。

## Phase 3: AI Command Center

目标：建立面向内部操作员的后台管理 Agent。

要做：

- Management Agent 对话会话
- 自然语言意图识别
- 产品动作计划生成
- 工具调用状态展示
- 高风险动作确认流
- 命令执行日志

第一版支持：

- 查询好友、群、账号、任务和日志
- 把好友加入或移出 Agent 运营
- 给指定好友发送消息
- 创建运营任务
- 生成朋友圈计划草稿
- 按当前账号列出 MCP 工具目录
- 通过后端代理执行 MCP 工具并写入审计日志

默认策略：

- 每个微信账号独立 MCP Key。
- 操作员在前端选择当前账号。
- LLM 可以选择当前账号完整 MCP 工具目录。
- 前端不直接调用 MCP，不展示 MCP Key 明文。

## Phase 4: 朋友圈运营

目标：建立内容计划、草稿、发布和历史记录。

要做：

- 朋友圈内容计划
- AI 草稿生成
- 素材选择
- 定时发布任务
- 发布历史
- 发布结果记录

默认策略：

- 先草稿，后发布。
- 发布必须有规则或人工确认。
- 不做随机自动发布。

## Phase 5: 微信群运营

目标：群消息分析、群画像和线索发现。

第一版要做：

- 群列表
- 群消息入库
- 群画像
- 群话题总结
- 重点消息识别
- 回复建议

暂不做：

- 自动群发言
- 自动群管理动作

## Phase 6: 多账号矩阵和数据分析

目标：支持多个微信账号协同运营。

要做：

- 多账号视图
- 账号运营状态
- 跨账号客户去重
- 模块级数据报表
- Agent 成效分析
- 失败率和响应时效监控

## Near-Term Engineering Tasks

建议最近开发顺序：

1. 给列表接口加分页和筛选。
2. 增加客户阶段变化日志。
3. 建内容资产最小模型。
4. 建 Agent Soul 和 Prompt 版本模型。
5. 把 Agent prompt 中的规则迁移到策略配置。
6. 做 AI Command Center 的最小闭环。
7. 增加 webhook 测试页面或 CLI。
8. 为 MCP/LLM 调用补 mock 测试。
