# Development Roadmap

## Phase 1: 用户运营闭环

目标：让私聊好友运营稳定、可控、可追踪。

已完成：

- Rust 后端
- React 后台
- MongoDB 数据层
- MCP 调用
- DeepSeek 模型调用
- 好友导入
- `normal / managed`
- 运营备注
- 画像生成
- 自动回复
- 跟进任务
- 日志记录

下一步建议：

- 用户标签
- 用户阶段
- 跟进频率
- 最近承诺记录
- 分页和搜索增强
- webhook 载荷样例和测试工具

## Phase 2: 内容资产 + Agent 策略

目标：让 Agent 的内容和行为边界可配置。

要做：

- 内容资产库
- FAQ/产品资料/案例库
- 品牌语气规则
- 禁用表达
- Agent 策略配置
- 频控规则
- 自动化边界配置

交付标准：

- Agent 回复能引用内容资产。
- 运营人员能配置 Agent 行为风格。
- 自动化边界不再硬编码在 prompt 里。

## Phase 3: 朋友圈运营

目标：建立内容计划、草稿、发布和历史记录。

要做：

- 朋友圈内容计划
- AI 草稿生成
- 素材选择
- 定时发布任务
- 发布历史
- 发布结果记录

默认策略：

- 先草稿，后发布。
- 发布必须有规则或人工确认。
- 不做随机自动发布。

## Phase 4: 微信群运营

目标：群消息分析、群画像和线索发现。

第一版要做：

- 群列表
- 群消息入库
- 群画像
- 群话题总结
- 重点消息识别
- 回复建议

暂不做：

- 自动群发言
- 自动群管理动作

## Phase 5: 多账号矩阵和数据分析

目标：支持多个微信账号协同运营。

要做：

- 多账号视图
- 账号运营状态
- 跨账号客户去重
- 模块级数据报表
- Agent 成效分析
- 失败率和响应时效监控

## Near-Term Engineering Tasks

建议最近开发顺序：

1. 补齐用户运营字段：标签、阶段、意向等级。
2. 给列表接口加分页和筛选。
3. 建内容资产最小模型。
4. 把 Agent prompt 中的规则迁移到策略配置。
5. 增加 webhook 测试页面或 CLI。
6. 为 MCP/LLM 调用补 mock 测试。
